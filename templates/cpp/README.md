# C++ Development

Modern C++ development environment with Conan package management and CMake build system.

## Philosophy

This template follows a clean separation of concerns:

- **Nix**: Provides development tools (Clang, CMake, Conan)
- **Conan**: Manages all project dependencies (fmt, spdlog, etc.)
- **CMake**: Builds your project with automatically generated presets

This is similar to how you'd use Nix for Python/OCaml tools while using `uv`/`opam` for packages.

## Features

- 🚀 **C++23** - Modern C++ standard
- 📦 **Conan 2** - Package management
- 🔧 **CMake Presets** - Consistent build configuration
- 🛠️ **LLVM Toolchain** - Latest compiler
- 🔍 **clang-tidy** - Static analysis
- ⚡ **ccache** - Build acceleration

## Configuration

Customize your environment in `flake.nix`:

```nix
config = {
  # Basic settings - always customize these
  basic = {
    name = "C++ Development";
    llvmVersion = "20";
    cppStandard = 23;
    compiler = "clang";
  };

  # Development tools - toggle features
  tools = {
    enableClangTidy = true;
    enableCcache = true;
    ccacheMaxSize = "5G";
    enableTesting = true;
    testFramework = "gtest";
  };

  # Advanced settings - optimization & performance
  advanced = {
    buildJobs = 12;
    warningLevel = "all";
    enableExceptions = true;
    enableRTTI = true;

    debug = {
      optimizationLevel = 0;
      enableSanitizers = true;
    };

    release = {
      optimizationLevel = 2;
      enableLTO = false;
      marchNative = false;
    };
  };
};
```


## Getting Started

```bash
# 1. Enter the directory (direnv loads Nix environment automatically)
cd your-project

# 2. Install dependencies
conan install . --profile=release --build=missing

# 3. Configure the build
cmake --preset=conan-release

# 4. Build the project
cmake --build --preset=conan-release

# 5. Run the application
./build/Release/myapp

# 6. Run the tests
ctest --preset=conan-release
```

### Debug Build

```bash
# For development with debug symbols and sanitizers
conan install . --profile=debug --build=missing
cmake --preset=conan-debug
cmake --build --preset=conan-debug
./build/Debug/myapp
ctest --preset=conan-debug
```

## How It Works

1. **Nix flake** provides LLVM toolchain, CMake, Conan
2. **shellHook** automatically:
   - Initializes git repository if needed
   - Sets up local `.conan2/` directory for Conan profiles
   - Links global Conan cache to avoid re-downloading packages
   - Generates debug and release Conan profiles with optimized flags
3. **Conan** installs dependencies into local cache
4. **CMake** uses Conan-generated presets for configuration
5. **clangd** (from Nix) uses `compile_commands.json` for IDE features

All tools use consistent compiler flags from the Nix configuration!

## Development Tips

1. **IDE Integration**: The environment generates `compile_commands.json` for clangd support
2. **Clean Build**: `rm -rf build/ && conan install . --profile=release --build=missing`
3. **Switch Profiles**: Use `--preset=conan-debug` or `--preset=conan-release`
4. **Faster Builds**: ccache enabled - subsequent builds will be faster

## Troubleshooting

**Conan packages not found:**
\`\`\`bash
conan remove "*" -c
conan install . --profile=release --build=missing
\`\`\`

**CMake preset not found:**
\`\`\`bash
ls CMakePresets.json  # Should exist after conan install
\`\`\`

**Build errors:**
\`\`\`bash
# Force rebuild dependencies
conan install . --profile=release --build=missing --build=<package>
\`\`\`
